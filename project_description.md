# 프로젝트: 초간단 회원제 댓글창 (Simple Auth Guestbook)

**A classic, server-rendered guestbook web application with user authentication.**

## Overview

이 프로젝트는 Python과 Flask를 사용하여 구현된 미니멀리즘 웹 애플리케이션입니다.
웹 개발의 가장 핵심적인 기능인 사용자 인증(회원가입, 로그인), 데이터베이스 연동, 동적 페이지 렌더링을 서버 사이드 렌더링(SSR) 방식으로 구현하여 웹의 근본적인 동작 원리를 명확하게 보여주는 것을 목표로 합니다.

누구나 댓글을 읽을 수 있지만, 오직 인증된 사용자만이 새로운 댓글을 작성할 수 있는 전통적인 게스트북(방명록) 기능을 제공합니다.

## Key Features

- **사용자 인증:** 안전한 회원가입 및 로그인 기능을 제공합니다.
- **비밀번호 보안:** 사용자의 비밀번호는 평문으로 저장되지 않고, 솔트(Salt)가 포함된 해시(Hash) 처리를 통해 안전하게 저장됩니다.
- **공개 댓글 목록:** 모든 방문자가 댓글 목록을 조회할 수 있습니다.
- **인증 기반 글쓰기:** 오직 로그인한 사용자만이 새로운 댓글을 작성할 수 있습니다.
- **프로필 확인:** 로그인한 사용자는 자신의 아이디와 이름 정보를 확인할 수 있는 개인 프로필 페이지를 가집니다.
- **클래식 UI:** 2000년대 초반의 웹 감성을 담은 깔끔하고 반응형인 사용자 인터페이스를 제공합니다.

## Tech Stack

| Category      | Technology        | Description
| :---          | :---              | :---    
| **Backend**   | Python            | 메인 프로그래밍 언어입니다.
|               | Flask             | 웹 서버의 라우팅과 핵심 로직을 처리하는 마이크로 웹 프레임워크입니다.
|               | Werkzeug          | 비밀번호 해싱 등 WSGI 유틸리티를 위해 사용되었습니다.
| **Frontend**  | HTML5             | 웹 페이지의 구조를 담당합니다.
|               | CSS3              | 웹 페이지의 스타일과 레이아웃을 담당합니다.
|               | Jinja2            | HTML에 데이터를 동적으로 렌더링하기 위한 Flask의 템플릿 엔진입니다.
| **Database**  | SQLite            | 별도의 서버가 필요 없는 파일 기반의 경량 데이터베이스입니다.

## Project Structure

프로젝트의 주요 폴더와 파일의 역할은 다음과 같습니다.

```
/
├── .venv/                      # 파이썬 가상 환경 및 프로젝트 의존성 관리 폴더
├── static/
│   └── style.css               # 애플리케이션 UI를 위한 메인 스타일시트
├── templates/
│   ├── base.html               # 공통 레이아웃(헤더, 푸터)을 담고 있는 기본 템플릿
│   ├── index.html              # 메인 페이지 (댓글 목록 및 작성 폼 표시)
│   ├── login.html              # 사용자 로그인 페이지
│   ├── profile.html            # 사용자 프로필 표시 페이지
│   └── register.html           # 사용자 회원가입 페이지
├── app.py                      # 라우팅, 핵심 로직, DB 연동을 처리하는 메인 Flask 애플리케이션
├── database.db                 # 사용자 및 댓글 데이터를 저장하는 SQLite 데이터베이스 파일
├── init_db.py                  # 데이터베이스 테이블을 생성하기 위한 초기화 스크립트
└── readme.md                   # 초기 프로젝트 명세서
```

## Getting Started

이 프로젝트를 로컬 환경에서 실행하기 위한 단계는 다음과 같습니다.

**Prerequisites:**
- Python 3.x

**1. 프로젝트 폴더로 이동**
```bash
cd path/to/SINE_web_project
```

**2. 가상 환경 생성 및 활성화**
```bash
# 가상 환경 생성 (최초 1회)
python -m venv .venv

# 가상 환경 활성화 (Windows)
.venv\Scripts\activate
```

**3. 의존성 설치**
```bash
pip install Flask Werkzeug
```

**4. 데이터베이스 초기화**
데이터베이스 파일과 테이블을 생성합니다. (프로젝트 설정 시 최초 1회만 실행)
```bash
python init_db.py
```

**5. 애플리케이션 실행**
```bash
python app.py
```

**6. 접속**
웹 브라우저를 열고 `http://127.0.0.1:5000` 주소로 접속합니다.

## Core Logic/Flow

1.  **회원가입:** 사용자는 `/register` 페이지에서 ID, 이름, 비밀번호를 입력합니다. 서버는 비밀번호를 해싱하여 `database.db`의 `User` 테이블에 새 사용자 정보를 저장합니다.
2.  **로그인:** 사용자는 `/login` 페이지에서 ID와 비밀번호를 입력합니다. 서버는 DB에 저장된 해시값과 입력된 비밀번호를 비교하여 인증을 수행합니다. 인증 성공 시, 사용자의 고유 ID를 **세션(Session)**에 저장하여 로그인 상태를 유지합니다.
3.  **댓글 조회 및 작성:**
    -   메인 페이지(`/`)에 접속하면, 서버는 `Comment` 테이블의 모든 댓글을 조회하여 화면에 렌더링합니다. 이 과정은 로그인 여부와 관계없이 모든 사용자에게 동일합니다.
    -   만약 사용자의 **세션**에 ID가 존재할 경우(로그인 상태), 서버는 댓글 작성 폼을 추가로 렌더링해줍니다.
    -   로그인한 사용자가 댓글을 작성하고 제출하면, 서버는 해당 내용을 세션 ID와 연결하여 `Comment` 테이블에 저장한 후, 다시 메인 페이지로 리디렉션합니다.
4.  **프로필 조회:** 로그인한 사용자가 `/profile` 페이지로 이동하면, 서버는 세션 ID를 이용해 사용자의 상세 정보를 DB에서 조회하여 화면에 보여줍니다.
