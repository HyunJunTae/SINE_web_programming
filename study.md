이 프로젝트에서 백엔드가 한 일 한눈에 보기
이 프로젝트의 백엔드는 크게 보면 7가지 역할을 합니다.

사용자가 어떤 URL로 들어왔는지 보고 알맞은 기능 실행
회원가입/로그인 폼 데이터를 받아 처리
로그인 상태를 세션으로 유지
사용자와 댓글 정보를 데이터베이스에 저장하고 읽기
로그인한 사람만 특정 기능을 쓰게 제어
조회한 데이터를 HTML 템플릿에 전달해 화면을 동적으로 구성
앱 시작과 데이터베이스 초기 구조를 준비
아래에서 각각을 예시와 함께 정리해볼게요.

1. URL 요청을 받아 기능으로 연결하는 역할
백엔드의 가장 기본 역할은 주소(URL)와 파이썬 함수를 연결하는 것입니다. 이 프로젝트에서는 Flask의 @app.route(...)가 그 역할을 합니다. 예를 들어 /는 메인 댓글 페이지, /register는 회원가입, /login은 로그인, /profile은 프로필 페이지, /logout은 로그아웃 기능으로 연결되어 있습니다. 즉, 사용자가 어떤 주소로 들어오느냐에 따라 백엔드가 서로 다른 동작을 수행합니다. app.py

예를 들면 이런 식입니다.

사용자가 "/" 접속 → index() 실행
사용자가 "/login" 접속 → login() 실행
사용자가 "/profile" 접속 → profile() 실행
이 말은 곧, 프론트의 화면 이동을 실제 기능으로 바꿔주는 중심이 백엔드라는 뜻입니다.

2. 폼 입력을 받아 실제 데이터를 처리하는 역할
프론트엔드의 HTML 폼은 입력창만 보여줄 뿐이고, 입력된 값이 실제로 유효한지 검사하고 저장하는 일은 백엔드가 합니다. 이 프로젝트에서는 회원가입, 로그인, 댓글 작성이 모두 그 구조로 되어 있습니다. register.html login.html index.html app.py

예시 1: 회원가입 처리
회원가입 폼에서 user_id, password, user_name을 입력하면, 백엔드는 이를 request.form[...]으로 받아옵니다. 그리고 값이 비어 있는지 확인한 뒤, 아이디 중복 여부를 검사하고, 문제가 없으면 DB에 저장합니다. app.py

즉 흐름은 이렇게 됩니다.

사용자가 회원가입 폼 작성
브라우저가 POST 요청 전송
백엔드가 입력값 받음
빈 값 검사
아이디 중복 검사
비밀번호 암호화
사용자 정보 저장
예시 2: 댓글 작성 처리
메인 페이지의 댓글 작성 폼은 로그인한 사용자에게만 보이고, 실제 등록은 index() 함수의 POST 처리에서 이루어집니다. 사용자가 댓글 내용을 보내면, 백엔드는 그 내용을 받아 DB에 저장합니다. index.html app.py

즉, **“입력 UI는 프론트엔드, 실제 처리와 저장은 백엔드”**라고 보면 됩니다.

3. 로그인 상태를 기억하는 역할
이 프로젝트 백엔드의 중요한 역할 중 하나는 사용자가 로그인했는지 기억하는 것입니다. 이것을 위해 Flask의 session을 사용하고 있습니다. 로그인 성공 시 session['user_id']와 session['user_name']를 저장하고, 로그아웃하면 session.clear()로 비웁니다. app.py

예시
로그인에 성공하면 백엔드는 다음과 같은 상태를 유지합니다.

session['user_id'] = 사용자 고유 번호
session['user_name'] = 사용자 이름
이 정보 덕분에 이후 요청에서도 “지금 접속한 사람이 누구인지”를 알 수 있습니다. 그래서:

헤더에 “안녕하세요, OOO님!” 표시 가능
댓글 작성 권한 확인 가능
프로필 페이지에서 내 정보 조회 가능
이 동작은 base.html의 네비게이션에도 바로 반영됩니다. 템플릿에서는 session.user_name이 있으면 로그인 상태용 메뉴를, 없으면 로그인/회원가입 메뉴를 보여줍니다. 즉, 세션은 백엔드가 관리하고, 화면은 그 결과를 보여주는 구조입니다. base.html

4. 데이터베이스와 연결해 저장·조회하는 역할
이 프로젝트 백엔드는 SQLite 데이터베이스를 이용해 회원 정보와 댓글을 영구 저장합니다. 단순히 변수에 담아두는 것이 아니라, 프로그램을 껐다 켜도 남는 구조입니다. 이를 위해 get_db()가 DB 연결을 만들고, 요청이 끝날 때 close_connection()이 연결을 닫습니다. app.py

데이터베이스 구조
init_db.py를 보면 두 개의 핵심 테이블이 있습니다.

User: 회원 정보 저장
Comment: 댓글 저장
그리고 Comment.user_id가 User.id를 참조하도록 외래 키가 걸려 있어서, 누가 어떤 댓글을 썼는지 연결됩니다. init_db.py

예시 1: 회원 저장
회원가입이 완료되면 백엔드는 다음 정보를 User 테이블에 넣습니다.

아이디
암호화된 비밀번호
표시 이름
예시 2: 댓글 저장
댓글 등록 시에는 현재 로그인한 사용자의 session['user_id']와 댓글 내용을 Comment 테이블에 저장합니다. app.py

예시 3: 댓글 조회
메인 페이지에서는 단순히 댓글만 읽는 게 아니라, Comment와 User를 JOIN해서 댓글 내용과 작성자 이름을 함께 가져옵니다. 그래서 화면에는 댓글 옆에 작성자 이름과 시간이 같이 표시됩니다. app.py index.html

이 부분은 백엔드가 단순 저장소가 아니라, 데이터를 조합해서 사용자에게 의미 있는 형태로 제공하는 역할을 한다는 좋은 예입니다.

5. 보안과 인증을 담당하는 역할
이 프로젝트 백엔드는 아주 기본적인 수준이지만 중요한 보안 역할도 하고 있습니다.

5-1. 비밀번호를 그대로 저장하지 않음
회원가입 시 generate_password_hash(password)를 사용해 비밀번호를 해시로 바꾼 뒤 저장합니다. 로그인할 때는 check_password_hash(...)로 비교합니다. 즉, DB에 평문 비밀번호를 저장하지 않습니다. app.py

이건 백엔드의 핵심 보안 책임 중 하나입니다.

5-2. 로그인한 사용자만 기능 사용 가능
댓글 작성 시 로그인하지 않은 사용자는 바로 저장하지 못하고 로그인 페이지로 이동합니다. 프로필 페이지도 로그인하지 않았다면 접근할 수 없도록 막아둡니다. app.py

예시
비로그인 사용자가 댓글 POST 요청 → 로그인 페이지로 리디렉션
비로그인 사용자가 /profile 접속 → 로그인 페이지로 리디렉션
즉, 권한 체크는 백엔드가 최종적으로 책임진다고 볼 수 있습니다. 프론트에서 버튼을 숨기는 것만으로는 충분하지 않고, 서버에서 실제 요청을 막아야 하기 때문입니다.

6. 화면에 필요한 데이터를 HTML에 주입하는 역할
이 프로젝트는 Jinja 템플릿을 사용하므로, HTML 파일이 완전히 고정된 문서가 아니라 백엔드가 데이터를 넣어 완성하는 동적 페이지입니다. base.html index.html profile.html

예시 1: 댓글 목록 렌더링
index()는 DB에서 comments를 조회한 뒤 render_template('index.html', comments=comments)로 넘깁니다. 그러면 템플릿의 {% for comment in comments %}가 반복되며 댓글 목록이 만들어집니다. app.py index.html

예시 2: 프로필 정보 표시
profile()은 DB에서 현재 사용자 정보를 읽어 render_template('profile.html', user=user)로 전달합니다. 그러면 템플릿에서 {{ user.user_id }}, {{ user.user_name }}로 출력합니다. app.py profile.html

예시 3: 공통 레이아웃 재사용
base.html은 공통 레이아웃이고, 다른 페이지들이 {% extends "base.html" %}로 상속합니다. 이때 로그인 상태에 따라 헤더 메뉴가 달라지는 것도 백엔드의 세션 데이터 덕분입니다. base.html login.html register.html

즉, 백엔드는 데이터를 만들고, 템플릿은 그 데이터를 화면으로 표현합니다.

7. 기본적인 검증과 예외 처리를 하는 역할
이 프로젝트 백엔드는 복잡하진 않지만 최소한의 검증과 오류 응답도 담당합니다. app.py

예를 들면:

댓글 내용이 없으면 400 반환
회원가입 필드가 비어 있으면 400 반환
이미 존재하는 아이디면 400 반환
로그인 실패 시 401 반환
세션에 사용자 ID는 있는데 DB에 사용자가 없으면 세션을 지우고 다시 로그인으로 보냄
이런 부분은 사용자 경험뿐 아니라 시스템 안정성에도 중요합니다. 즉, 백엔드는 단순히 “성공 처리”만 하는 게 아니라, 잘못된 요청을 걸러내고 안전하게 대응하는 역할도 합니다.

8. 프로젝트 초기 세팅을 준비하는 역할
init_db.py는 사용자가 직접 보는 기능은 아니지만, 백엔드 관점에서 매우 중요합니다. 이 파일은 서비스가 동작하기 전에 필요한 데이터 구조를 미리 만드는 역할을 합니다. init_db.py

여기서 하는 일은 다음과 같습니다.

database.db 생성 또는 연결
User 테이블 생성
Comment 테이블 생성
댓글 작성 시각이 자동 저장되도록 timestamp DEFAULT CURRENT_TIMESTAMP 설정
댓글과 사용자 관계를 외래 키로 연결
즉, app.py가 실제 서비스 운영 로직이라면, init_db.py는 서비스를 굴릴 수 있게 바닥을 깔아주는 준비 작업이라고 볼 수 있습니다.
