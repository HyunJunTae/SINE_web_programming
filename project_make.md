# 프로젝트 명세서: 초간단 회원제 댓글창 (Simple Auth Guestbook)

## 1. 프로젝트 목표
* 가장 대중적이고 배우기 쉬운 기술 스택을 활용하여, 기본적인 회원가입/로그인 기능과 누구든 볼 수 있지만 로그인한 사용자만 글을 쓸 수 있는 단일 댓글창 웹사이트를 구축한다.

## 2. 추천 기술 스택 (입문자 최적화 조합)
* **프론트엔드 (Frontend):** * HTML/CSS/JS (Vanilla) - 프레임워크 없이 생으로 구현하여 웹의 기본 동작(Form 전송, DOM 조작)을 명확히 이해하도록 함.
* **백엔드 (Backend):** * Python + Flask - 코드가 직관적이고 설정이 매우 간단하여 입문에 최적화됨.
* **데이터베이스 (Database):** * SQLite - 별도의 DB 서버 설치 없이 파일(.db) 하나로 관리할 수 있어 입문에 최적화됨.
* **서버/배포 (Deployment - 추후 단계):** * Ubuntu (AWS 또는 Oracle Cloud) + Nginx

## 3. 핵심 기능 및 사용자 흐름 (User Flow)

### 3.1. 회원가입 및 로그인 (Authentication)
* **기능:** 사용자 정보를 DB에 저장하고, 로그인 시 대조하여 인증을 수행한다.
* **사용자 흐름:**
    1.  사용자는 회원가입 페이지에서 ID, 비밀번호, 이름을 입력하여 가입한다.
    2.  로그인 페이지에서 ID와 비밀번호를 입력한다.
    3.  로그인 성공 시 세션(Session)을 통해 사용자 상태를 기억하고 메인 화면으로 이동한다.
* **보안 사양:** *중요: 비밀번호는 DB에 평문으로 저장하지 않고, 반드시 해시(Hash) 처리하여 저장해야 한다.* (예: Werkzeug의 security 라이브러리 활용)

### 3.2. 메인 화면 및 댓글창 (Main Page & Guestbook)
* **기능:** 딱 하나의 단일 댓글창(방명록 형식)이 존재하며, 모든 글의 목록을 보여준다.
* **작성 권한:** 누구든 글을 볼 수 있지만, 로그인한 사용자만 글을 쓸 수 있다.
* **사용자 흐름:**
    * **비로그인 사용자:** 댓글 목록을 볼 수는 있지만, 글 작성 폼 대신 "로그인 후 글을 쓸 수 있습니다"라는 안내와 로그인 버튼을 본다.
    * **로그인 사용자:** 댓글 목록 하단에 글 작성 폼(Textarea, Submit 버튼)을 본다.
    * **글 작성 시:** 로그인한 사용자의 '이름'과 '글 내용'이 DB에 저장되고, 메인 화면의 댓글 목록에 즉시 반영되어 쭉 보인다.

## 4. 데이터베이스 스키마 (Draft)

### 4.1. `User` 테이블
* `id` (PK, Integer, AutoIncrement)
* `user_id` (String, Unique, 필수)
* `password_hash` (String, 필수) - 해시 처리된 비밀번호
* `user_name` (String, 필수) - 실제 화면에 표시될 이름

### 4.2. `Comment` 테이블
* `id` (PK, Integer, AutoIncrement)
* `user_id` (FK to User, Integer, 필수) - 글을 쓴 사용자
* `content` (Text, 필수) - 글 내용
* `timestamp` (DateTime, Default=Current) - 작성 시간

## 5. UI/UX Vibe (바이브 코딩 가이드)
* **컨셉:** 90년대 후반~2000년대 초반의 '클래식 웹' 감성을 현대적으로 재해석한 깔끔함.
* **색상:** 화이트 배경에 블랙 텍스트, 포인트 컬러는 부드러운 파란색 (#3498db).
* **레이아웃:** 중앙 집중식, 반응형 (모바일에서도 가독성 확보).
* **기타:** 글 작성 폼은 Textarea를 사용하여 여러 줄 작성을 지원함.

## 6. 단계별 구현 로드맵 (초기 에이전트 미션)

### [1단계] 로컬 환경에서 기능 구현 (본 미션)
먼저 본인의 컴퓨터에서 웹사이트가 돌아가게 만드는 것을 목표로 한다. Gemini Agent는 아래 순서대로 파일을 생성하고 코드를 작성해야 한다.

1.  **백엔드 기본 구성:** Flask를 설치하고 `app.py`에서 Hello World를 띄우는 기본 구조를 만든다.
2.  **DB 스키마 설계:** SQLite를 사용하여 `User` 테이블과 `Comment` 테이블을 생성하는 스크립트(`init_db.py` 등)를 만든다.
3.  **회원가입/로그인 로직:** * 사용자가 입력한 ID/PW를 DB에 저장하고, 로그인 시 대조하는 기능을 `app.py`에 구현한다.
    * 비밀번호 해시(Hash) 처리를 반드시 적용한다.
4.  **세션 관리:** 로그인 성공 시 사용자를 어떻게 기억할지(Flask Session 활용) 구현한다.
5.  **메인 댓글창 로직:**
    * 로그인 상태에 따라 글 작성 폼 노출 여부를 결정한다.
    * 글 작성 시 로그인된 사용자 ID와 연동하여 DB에 저장하고, 목록을 불러와 화면에 뿌려준다.
6.  **프론트엔드 연결:** HTML Form 태그와 Vanilla JS를 이용해 백엔드(`app.py`)와 데이터를 주고받는 HTML 파일을 작성한다.
